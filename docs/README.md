# Documentation

This directory contains project documentation, guides, and references.

Start here:
1. `docs/usage.md` for the end-user workflow.
2. `docs/file-formats.md` for config/report schemas.
3. `docs/architecture.md` for module and data-flow details.
4. `docs/troubleshooting.md` for common build/runtime issues.

## Index
- `docs/usage.md` - end-user workflow for the GUI.
- `docs/file-formats.md` - configuration and report schemas with examples.
- `docs/architecture.md` - module and data-flow overview.
- `docs/development.md` - build options, test notes, and contributor tips.
- `docs/troubleshooting.md` - common build and runtime issues.
- `docs/references/` - bibliographic references and supporting material.
- `docs/legacy/` - archived coursework artifacts from earlier project revisions.
